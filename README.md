# Hey, I'm Masab Farooque

I build web applications and AI systems. Based in Pakistan, working with clients worldwide.

## What I Do

I'm a full-stack developer who specializes in two main areas: building modern web applications and integrating AI into products. Most of my work involves Next.js on the frontend, Node.js or FastAPI on the backend, and various LLMs when intelligence is needed.

### Web Development

I work primarily with the React ecosystem. Next.js is my go-to framework for anything production-facing because of its flexibility with SSR, SSG, and API routes. On the backend, I switch between NestJS when I need structure and type safety, and FastAPI when I need speed and Python's ecosystem.

**My typical stack looks like:**

- **Frontend:** Next.js 14+, React 18+, TypeScript, Tailwind CSS
- **Backend:** NestJS, Node.js, FastAPI, Express
- **Databases:** PostgreSQL for relational data, MongoDB when flexibility matters, Redis for caching
- **Deployment:** Vercel for Next.js, Docker for everything else, AWS when needed

I care about performance. That means proper code splitting, lazy loading, optimized images, and not shipping unnecessary JavaScript. I also care about maintainability, so I write tests, document decisions, and keep dependencies up to date.

### AI Engineering

The AI space moves fast, and I've been keeping up. I build applications that use LLMs in practical ways - chatbots that actually help users, document analysis tools, content generation systems, and automation workflows.

**What I work with:**

- **LLM APIs:** OpenAI (GPT-4, GPT-4o), Anthropic (Claude), Groq, OpenRouter
- **Frameworks:** LangChain, LangGraph, LlamaIndex
- **Vector Databases:** Pinecone, FAISS, Chroma
- **Techniques:** RAG (Retrieval-Augmented Generation), fine-tuning, prompt engineering, function calling

I've built RAG systems that pull from thousands of documents, conversational agents that maintain context across long conversations, and automation tools that use LLMs to make decisions. The key is knowing when to use AI and when a simple regex will do.
